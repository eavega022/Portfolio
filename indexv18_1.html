PROJECT 01 · POWER BI + EXCEL

Customer Satisfaction Analysis — Bilingual Contact Center

End-to-end analytics project simulating a real bilingual call center environment (EN/ES). Starting from a raw, intentionally dirty dataset, the project covers data cleaning, exploratory analysis, metric calculation, and the construction of an interactive Power BI dashboard — replicating the analytical work performed at Teleperformance where CSAT was improved from 75% to 83%.

DATASET
2,760 records · 14 variables
PERIOD
Jan – Dec 2023
TOOLS
Excel · Power BI · Power Query
DOMAIN
CX Analytics · BPO
LANGUAGES
EN (1,231) · ES (1,529)
AGENTS TRACKED

//·OBJECTIVES

What this project answers

01
Track CSAT evolution
Measure how customer satisfaction changed month by month throughout 2023 and identify inflection points.
02
Identify root causes of dissatisfaction
Determine which channels, shifts, issue types, and resolution outcomes have the biggest negative impact on CSAT.
03
Compare performance segments
Benchmark agents, languages, and time slots to surface the top and bottom performers and understand the gap.
04
Deliver actionable recommendations
Translate data insights into concrete operational decisions that a CX team can implement immediately.
//·METHODOLOGY

How the analysis was built

1
Data Generation & Raw Dataset
A realistic 2,860-row dataset was constructed simulating a bilingual contact center, incorporating intentional data quality issues: 60 duplicates, 40 null CSAT values, 30 inconsistent date formats, 30 dirty channel entries, and 20 negative wait times.
Excel2,860 raw rows
2
Data Cleaning (5-Step Process)
Using Excel native tools: removed duplicates via Data → Remove Duplicates, deleted null CSAT rows via filter, corrected date formats with Text to Columns, normalized channel values using PROPER(TRIM()) formula + Paste Special Values, and converted negative wait times to absolute values.
PROPER(TRIM())Pivot Table audit2,760 clean rows
3
Exploratory Analysis & KPI Definition
Defined key metrics: Average CSAT Score, % Satisfied Clients (score ≥ 4), resolution rate, average wait and handle time. Segmented by channel, language, shift, agent, and issue type. Used Excel pivot tables for initial validation before moving to Power BI.
KPI DesignSegmentationPivot Tables
4
Dashboard Construction in Power BI
Built a 2-page interactive dashboard covering monthly trend, satisfaction by channel and language, dissatisfied clients by issue type, scatter analysis of wait time vs CSAT, top 5 agents table, and distribution by resolution type. Includes slicer for dynamic filtering.
Power BI Desktop2 Pages14 VisualsSlicer
5
Insight Synthesis & Recommendations
Translated visual patterns into 6 actionable business recommendations targeting the root causes of low satisfaction. Each recommendation is supported by specific data points from the dashboard.
6 RecommendationsData Storytelling

//·METHODOLOGY

How the analysis was built

1
Database Setup — CSV Import
The Clean_Data sheet from Project 1 was exported as CSV (UTF-8) from Excel and imported into DB Browser for SQLite as a table named csat_data. 4 NULL rows were detected and removed, confirming exactly 2,760 clean records.
DB Browser for SQLiteCSV ImportDELETE WHERE NULL
2
Date Parsing Fix — INSTR + SUBSTR
The Fecha column was stored in M/D/YYYY format. The fix used CAST(SUBSTR(Fecha, 1, INSTR(Fecha, '/') - 1) AS INTEGER) to correctly extract month numbers, producing clean 12-row monthly aggregations.
INSTR()SUBSTR()CAST AS INTEGER
3
8 Analytical Queries — Full Coverage
Queries covered record verification, distinct value validation, overall CSAT with CASE WHEN, monthly trend, channel/shift/language breakdowns, top & bottom agent rankings with LIMIT, issue-type dissatisfaction, and resolution impact — all with inline comments.
GROUP BYORDER BYROUND(AVG())CASE WHENLIMIT
4
Results Documented as .sql File
All queries saved as CSAT_Analysis_Queries.sql with section comments — standalone evidence of SQL proficiency, reviewable without any database tool.
CSAT_Analysis_Queries.sqlDocumented

PROJECT 03 · PYTHON + POWER BI · OPEN DATA

3.75 Million Real Complaints from Colombia's Financial System

Using open data from Colombia's Superintendencia Financiera, this project analyzes every complaint filed between 2015 and 2021. The full pipeline was built in Python (Pandas): loading, cleaning 223,405 dirty records, exploratory analysis, and export to Power BI. The dashboard reveals that only 23% of financial complaints are ever resolved.

SOURCE
Superintendencia Financiera
RECORDS
3,754,916 original · 3,260,851 clean
PERIOD
2015 – 2021
VARIABLES
8 columns · 676 entities · 124 products
TOOLS
Python · Pandas · Power BI
DOMAIN
Financial Services · Open Data
//·OBJECTIVES

What this project answers

01
Handle a production-scale dataset with Python
Process 3.75M records using Pandas — demonstrating ability to work with datasets too large for Excel, with real dirty data from a government source.
02
Identify which banks and products fail consumers most
Rank entities by total complaints and resolution rate — separating high-volume banks from genuinely poor performers.
03
Measure the impact of the COVID-19 pandemic
Quantify the 2020 spike in complaints and determine whether the system recovered in 2021 using YoY trend analysis.
04
Build a complete Python-to-Power BI pipeline
Export 6 aggregated CSV tables from Python and import to Power BI — demonstrating end-to-end data engineering from raw source to executive dashboard.
//·METHODOLOGY

How the analysis was built

1
Data Loading & Initial Exploration
414MB CSV loaded with pd.read_csv(low_memory=False). Initial inspection revealed ANIO stored as text with commas ('2,016'), mixed types, and 306K null values in PRODUCTO and MOTIVO.
Pandasread_csv()dtypesvalue_counts()
2
Data Cleaning — 6 Issues Resolved
6 cleaning operations: ANIO text-to-int conversion, removal of 220,837 PRUEBA test records, 2,565 null entity rows, dirty TIPO_ENTIDAD values, text normalization, and 'NAN' string replacement. Final: 3,260,851 clean records.
str.replace()pd.to_numeric()Boolean filteringstr.upper()
3
Handling Incomplete Years (2022–2023)
Year audit revealed 2022 data ends in October (48K → 1.7K records/month) and 2023 had only 4,118 records. Both excluded. Final analytical dataset: 2015–2021, 7 complete years.
groupby()Boolean maskData completeness audit
4
Exploratory Analysis & Resolution Rate Calculation
Calculated complaints by entity, product, motive, resolution rate per entity (FINALIZADOS/total via lambda aggregation), and YoY evolution for top 5 banks. Resolution range: 18.8% (worst) to 32.8% (best).
groupby().agg()lambdavalue_counts()YoY analysis
5
Export Pipeline → Power BI Dashboard (3 pages)
6 CSVs exported to Power BI: 3-page dashboard covering KPIs, trend line, top banks, product/motive breakdowns, resolution rate table, and 6 narrative insight cards.
to_csv()Power BIDAX measure3 pages · 14 visuals
